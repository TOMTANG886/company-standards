# Checklist

**PR/Spec:** · **Date:** · **Author:** @

## Author

- [ ] Frontmatter complete (`id`, `status: review`, `version` bumped, `updated` set)
- [ ] Required sections present and non-empty
- [ ] Acceptance criteria are testable (no vague language)
- [ ] `npm run validate` passes

## Reviewer

- [ ] Problem statement accurate
- [ ] Solution technically sound
- [ ] Acceptance criteria unambiguous and sufficient
- [ ] Open questions resolved or deferred with owner + date

## Deploy

- [ ] Staging verified
- [ ] Rollback plan understood by on-call
- [ ] Monitoring in place
- [ ] Feature flag off by default
- [ ] Success metrics confirmed post-deploy
